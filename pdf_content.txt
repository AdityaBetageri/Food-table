1. Project Overview
  TableTapрџ’Ў is a cloud-based Smart QR Restaurant Ordering System that allows hotel and 
restaurant owners to digitise their entire ordering workflow вЂ” from table to kitchen to billing 
вЂ” without any physical menus or paper.
Customer scans a QR в†’ browses digital menu в†’ places order в†’ kitchen receives it 
instantly в†’ bill is auto-generated.
1.1  Problem Statement
Traditional restaurant systems rely on handwritten order slips, physical menus, and manual 
communication between waiters and kitchen staff. This leads to order errors, delays, unhygienic menu 
sharing, and poor customer experience вЂ” especially during peak hours.
1.2  Solution
TableTap replaces all of this with a seamless, real-time digital system. Each table gets a unique QR 
code. Customers self-order, the kitchen dashboard receives it instantly via WebSocket, and the admin 
has full visibility and analytics вЂ” all from a single unified platform.
1.3  Key Highlights
вЂўSign-up & onboarding is exclusively for Hotel/Restaurant Owners
вЂўSingle unified dashboard вЂ” owner and chef management combined
вЂўPublic-facing Landing Page showcasing the platform's amenities and features
вЂўReal-time order flow using Socket.io вЂ” zero page refresh required
вЂўQR code auto-links to a mobile-first customer menu with table context
вЂўRole-based access: Owner, Chef, Waiter, Cashier вЂ” all under one hotel account

9.2  Socket Room Strategy
To avoid broadcasting all events to all clients (which wastes bandwidth and exposes data), every 
connected client joins a private room identified by their hotel ID.
// On client connection:
socket.join(`hotel_${hotelId}`);
// On server emit (only to that hotel's clients):
io.to(`hotel_${hotelId}`).emit('new_order', orderData);
